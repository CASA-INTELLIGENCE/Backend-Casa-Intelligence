## 🚀 Quick Start

### Step 1: Install Dependencies

```bash
cd backend
pip install -r requirements.txt
```

**New dependencies added:**
- `cryptography>=42.0.5` - For encryption
- `python-json-logger>=2.0.7` - For structured logging

### Step 2: Run Security Setup

```bash
python setup_security.py
```

This will:
1. Create `.env` from `.env.example` if needed
2. Generate a secure `ENCRYPTION_KEY`
3. Add it to your `.env` file
4. Show you what credentials to update

### Step 3: Update Your `.env` File

Edit `backend/.env` and set:

```env
# CRITICAL - Already generated by setup script
ENCRYPTION_KEY=<44-character-key-already-set>

# REQUIRED - Get from https://aistudio.google.com/app/apikey
GEMINI_API_KEY=your_new_gemini_key_here

# REQUIRED - Your router password
ROUTER_PASSWORD=your_router_password_here

# OPTIONAL - For Alexa TTS
AMAZON_EMAIL=your_email@example.com
AMAZON_PASSWORD=your_amazon_password
```

### Step 5: Verify Git Protection

```bash

  'git rm --cached --ignore-unmatch backend/.env' \
  --prune-empty --tag-name-filter cat -- --all
```

## 🔍 How It Works

### Token Encryption

**Before:**
```
samsung_token.txt
-----
abc123def456ghi789...  <-- PLAIN TEXT
```

**After:**
```
.credentials/samsung_tv.json
-----
{
  "token": "gAAAAABh..." <-- ENCRYPTED with Fernet
}
```

**Access:**
```python
from security import cred_manager

# Store
cred_manager.store_credential("samsung_tv", "token", "abc123...")

# Retrieve
token = cred_manager.get_credential("samsung_tv", "token")
```

### Log Sanitization

**Before:**
```
INFO: Router connected with token=abc123def456...
INFO: User email: user@example.com
```

**After:**
```
INFO: Router connected with token=***REDACTED***
INFO: User email: ***EMAIL***
```

**Patterns Detected:**
- API Keys: `AIza...` → `***GEMINI_KEY***`
- Tokens: `token=xyz` → `token=***REDACTED***`
- Emails: `user@domain.com` → `***EMAIL***`
- Passwords: `password=xyz` → `password=***`

### Configuration Validation

**On startup, the system checks:**
- ✅ `ENCRYPTION_KEY` exists and is 44 characters
- ✅ `GEMINI_API_KEY` exists and starts with "AIza"
- ✅ `ROUTER_PASSWORD` is not empty
- ✅ `SAMSUNG_TV_IP` is valid IP format (if provided)

**If validation fails:**
```
============================================================
ERROR: Missing required configuration: ENCRYPTION_KEY
Help: Generate with: python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
============================================================

Please check your .env file and fix the above errors.
```

## 🛡️ Security Best Practices

### ✅ DO

1. **Keep `.env` local** - Never commit to git
2. **Rotate credentials** if they were exposed
3. **Use strong passwords** - Avoid personal info in passwords
4. **Review logs** regularly for suspicious activity
5. **Update dependencies** monthly for security patches

### ❌ DON'T

1. **Don't hardcode credentials** in source code
2. **Don't share .env files** via chat/email
3. **Don't use the same password** across services
4. **Don't commit sensitive files** (.credentials/, *.enc)
5. **Don't log credentials** even partially
